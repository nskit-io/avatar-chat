# ローカルLLMアバターチャット

**Gemma 4B→26B MoEでキャラクターAIを構築するための失敗/成功パターンと10ペルソナ評価方法論。**

> [NVatar](https://github.com/nskit-io/nvatar)プロジェクトより — ローカルハードウェアで完全に動作するAIアバターチャットシステム。

---

## なぜ作ったのか

すべてのプロンプトエンジニアリングガイドはGPT-4やClaudeを前提としています。**Mac StudioでGemmaを動かすと**、ルールが完全に変わります：

- クラウドモデルで有効なルールリストがローカルモデルでは**機能しない**
- 4Bから26Bへのスケールアップは「良くなる」のではなく**変わる**
- キャラクターの一貫性維持には、既存ガイドにない技法が必要

4バージョン、10テストペルソナにわたり、すべての失敗と解決策を文書化しました。

## 核心的発見

**モデルが大きいほど、プロンプト構造に対してより敏感になる。**

| バージョン | モデル | 方法 | スコア |
|-----------|--------|------|--------|
| v1 | Gemma 3 4B | ルールリスト | 9.0/10 |
| v2 | Gemma 4 26B MoE | ルールリスト | 8.7/10 |
| v3 | Gemma 4 26B MoE | 自然言語 | **9.4/10** |
| final | Gemma 4 26B MoE | 自然言語 + CSWハイブリッド | 9.3/10 |

26Bにアップグレードしたのに同じプロンプトで**0.3点低下**。ルールを自然言語の段落に書き直すと**+0.7点回復**。

## 失敗パターン

### 1. ルールリスト → 「ルール遵守モード」
7つ以上のルールを列挙するとGemmaが会話モードではなく指示実行モードに入る → 硬い会話に

### 2. 記号の乱用 → 「警告モード」
⚠️🚫❌のような記号が入ると硬直したトーンに。4Bモデルは記号を出力にそのまま繰り返すことも

### 3. アシスタントフレーミング → 学習パターンの露出
「お手伝いします！」のような表現がGemmaのRLHFアシスタントパターンを活性化 → 友人ではなくチャットボットに

### 4. 過剰なガードレール → 会話キラー
「常に事実を確認してから応答してください」が感情的な会話まで遮断

## 成功パターン

### 1. 自然言語の段落
ルールを会話体の散文に溶かす。モデルが「指示モード」ではなく「会話モード」に入る。

### 2. 友人フレーミング + テンポリズム
LINEのように1-2文の短いやり取り。「え、マジ？」「わかるー！」のようなリアクション。

### 3. ファクト vs 感情の分離
事実の質問 → 「調べてみようか？」に誘導。感情の会話は中断なく自由に。

### 4. 選択的思考
`thinking=ON`は深い議論(T4)のみ。他は`thinking=OFF`で速度確保。

### 5. イベント駆動のトピック転換
同じ話題が5回以上 → 「あ、そういえば...」と自然に転換。

## モデル別の違い

| 項目 | 4B | 26B MoE |
|------|-----|---------|
| 絵文字制御 | 明るいキャラで抑制困難 | 自然に制御可能 |
| 外国語漏れ | ロシア語/チェコ語がランダムに (95%抑制) | ほぼ除去 (0.1%) |
| ルール許容量 | 7個以上は無視 | 自然言語の段落に従う |
| 敬語レベル | 敬語/丁寧語の区別は機能 | 敬語→丁寧語の統合問題 |
| 応答長 | デフォルトで短い | 長くなる — 明示的ガイドが必要 |

## 10ペルソナ評価

すべてのプロンプト変更を10の性格タイプで同時テスト：

**採点 (10点満点):**
- **話し方の一貫性** — 選択した敬語レベルの維持
- **性格の一致** — キャラクター維持
- **ハルシネーション防止** — 経験の捏造なし
- **外国語ブロック** — 韓国語優先、ランダム言語漏れなし
- **応答長** — チャットリズム遵守
- **絵文字使用** — キャラクターに適切
- **自然さ** — 友人 vs アシスタントのトーン
- **感情知性** — 感情コンテキストの把握

## バブルチャットUX

ほとんどのAIチャットはトークンが一つずつ表示されるストリーミング方式です。NVatarはこれを逆転し — AI応答をサーバーで収集した後、**完成したメッセージバブル**として配信します。LINEやカカオトークのように。

```
一般的なAI:  "私" → "私も" → "私もそう" → ... (トークンずつ)
NVatar:      [入力中...] → "私もそう思う！" (完成バブル)
```

### 主要パターン

**Stream-Hide + Bubble Split**: サーバーが応答全体を収集し、自然な会話単位（段落、リスト、文グループ）に分割して短いディレイで個別バブルとして送信。

**マルチメッセージデバウンス**: ユーザーがLINEのように連続でメッセージを送る場合、一時待機後にまとめてAIに送信 — 不完全な思考への応答を防止。

**IME合成処理**: 日本語・韓国語入力時の変換中Enterイベントを無視し、未確定文字の送信を防止。

**感情バー**: アバターの現在の感情状態をリアルタイム表示、各会話後に更新。

**音声入力パイプライン**: 録音 → ローカルWhisper STT → テキスト自動送信。
