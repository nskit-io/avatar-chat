# 로컬 LLM 아바타 채팅

**Gemma 4B→26B MoE에서 캐릭터 AI를 만들기 위한 실패/성공 패턴과 10인 페르소나 평가 방법론.**

> [NVatar](https://github.com/nskit-io/nvatar) 프로젝트에서 추출 — 로컬 하드웨어에서 완전히 구동되는 AI 아바타 채팅 시스템.

---

## 왜 만들었나

모든 프롬프트 엔지니어링 가이드는 GPT-4나 Claude를 전제합니다. **Mac Studio에서 Gemma를 돌리면** 규칙이 완전히 달라집니다:

- 클라우드 모델에서 잘 되는 규칙 리스트가 로컬 모델에서는 **망함**
- 4B에서 26B로 올리면 그냥 "좋아지는" 게 아니라 **달라짐**
- 캐릭터 일관성 유지에는 어떤 가이드에도 없는 기법이 필요

4개 버전, 10개 테스트 페르소나에 걸쳐 모든 실패와 해결책을 문서화했습니다.

## 핵심 발견

**모델이 클수록 프롬프트 구조에 더 민감하다.**

| 버전 | 모델 | 방법 | 점수 |
|------|------|------|------|
| v1 | Gemma 3 4B | 규칙 리스트 | 9.0/10 |
| v2 | Gemma 4 26B MoE | 규칙 리스트 | 8.7/10 |
| v3 | Gemma 4 26B MoE | 자연어 | **9.4/10** |
| final | Gemma 4 26B MoE | 자연어 + CSW 하이브리드 | 9.3/10 |

26B로 업그레이드했는데 같은 프롬프트로 **0.3점 하락**. 규칙을 자연어 문단으로 재작성하니 **+0.7점 회복**.

## 실패 패턴

### 1. 규칙 리스트 → "규칙 따르기 모드"
7개 이상 규칙을 나열하면 Gemma가 대화 모드가 아닌 지시 수행 모드로 진입 → 딱딱한 대화

### 2. 기호 남용 → "경고 모드"
⚠️🚫❌ 같은 기호가 들어가면 경직된 톤으로 전환. 4B 모델은 기호를 출력에 그대로 반복하기도 함

### 3. 어시스턴트 프레이밍 → 학습 패턴 노출
"도와줄게!" 같은 표현이 Gemma의 RLHF 어시스턴트 패턴을 활성화 → 친구가 아닌 챗봇이 됨

### 4. 과도한 가드레일 → 대화 킬러
"항상 사실을 확인한 후 응답하세요"가 감정적 대화까지 차단

## 성공 패턴

### 1. 자연어 문단
규칙을 대화체 산문으로 풀어쓰기. 모델이 "지시 모드" 대신 "대화 모드"로 진입.

### 2. 친구 프레이밍 + 틱택 리듬
카카오톡처럼 1-2문장 짧은 교환. "아 진짜?", "나도 그래!" 같은 리액션.

### 3. 팩트 vs 감정 분리
사실 질문 → "찾아볼까?" 유도. 감정 대화는 중단 없이 자유롭게.

### 4. 선택적 사고
`thinking=ON`은 심층 토론(T4)에서만. 나머지는 `thinking=OFF`로 속도 확보.

### 5. 이벤트 기반 화제 전환
같은 주제 5회 이상 → "아 그러고 보니..." 자연스러운 전환.

## 모델별 차이

| 항목 | 4B | 26B MoE |
|------|-----|---------|
| 이모지 제어 | 밝은 캐릭터에서 억제 어려움 | 자연스럽게 제어 가능 |
| 외국어 누출 | 러시아어/체코어 랜덤 (95% 억제) | 거의 제거 (0.1%) |
| 규칙 수용력 | 7개 이상 무시 | 자연어 문단 잘 따름 |
| 존대 수준 | 존대/반존대 구분 작동 | 존대→반존대 병합 이슈 |
| 응답 길이 | 기본적으로 짧음 | 길어짐 — 명시적 가이드 필요 |

## 10인 페르소나 평가

모든 프롬프트 변경을 10개 성격 유형에 동시 테스트:

**채점 (10점 만점):**
- **말투 일관성** — 선택한 존대 수준 유지
- **성격 일치** — 캐릭터 유지
- **환각 방지** — 경험 날조 안 함
- **외국어 차단** — 한국어 우선, 랜덤 언어 누출 없음
- **응답 길이** — 카톡 리듬 준수
- **이모지 사용** — 캐릭터에 적합
- **자연스러움** — 친구 vs 어시스턴트 톤
- **감정 지능** — 감정 맥락 파악

**교차 대화 테스트** 5개 성격 쌍 (밝음↔차가움, 유머↔진지 등)에서 **차가운 캐릭터가 밝은 캐릭터에 동화되는 현상** 발견 — Gemma의 기본 긍정 편향.

## 시스템 프롬프트 구조

시스템 프롬프트는 정체성, 톤, 대화 스타일, 기억 컨텍스트, 안전을 다루는 자연어 문단으로 구성 — 구체적 구조는 비공개.

## 버블 채팅 UX

대부분의 AI 채팅은 토큰이 하나씩 나타나는 스트리밍 방식입니다. NVatar는 이를 뒤집어 — AI 응답을 서버에서 수집한 후 **완성된 말풍선**으로 전달합니다. 카카오톡이나 LINE처럼.

```
일반 AI:   "나" → "나도" → "나도 그" → "나도 그래" → ... (토큰씩)
NVatar:    [입력중...] → "나도 그래!" (완성된 버블)
```

### 핵심 패턴

**Stream-Hide + Bubble Split**: 서버가 전체 응답을 수집하고, 자연스러운 대화 단위(문단, 리스트, 문장 그룹)로 분할하여 짧은 딜레이를 두고 개별 버블로 전송.

**멀티 메시지 디바운싱**: 사용자가 카톡처럼 연속으로 메시지를 보낼 때, 잠시 대기 후 하나로 합쳐서 AI에 전달 — 불완전한 생각에 응답하는 것을 방지.

**IME 조합 처리**: 한글/일본어 입력 시 조합 중 Enter 이벤트를 무시하여 미완성 글자 전송을 방지.

**감정 바**: 아바타의 현재 감정 상태를 실시간 표시, 매 대화 후 갱신.

**음성 입력 파이프라인**: 녹음 → 로컬 Whisper STT → 텍스트 자동 전송.
